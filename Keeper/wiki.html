Welcome to Keeper's Wiki

Inside this wiki users can find everything from installing to compiling keeper into their project. Sometimes wiki gets updated after few days of a update being pushed however it won't be always late for the wiki to be updated.Please refer to https://www.services.keeper.net/ to view the status of all our backend services as well as the front end services.

In case there has been a problem accessing our wiki, users can always save some parts of the wiki offline and access them whenever they like. Pro subscription is recommended for saving wiki blocks offline.

Requirements for keeper are pretty simple nothing fancier just that it runs primarly on python and requires it totally inside your API for web app as well. Keeper allows your data from web applications to be stored in smaller files. To install keeper, you can download the zip folder, un-zip it and then simply put it inside your project folder under assets/keeper. If you don't have a assets/keeper folder created then make one first. After these steps simply add the code in all the HTML files you like the data to be protected for {{keeper.access.activate.on}} and keeper will take care of them all.

Functions and Framework of keeper is really easy-to-use, data is hard to handle and it's also sensitive, that is why keeper stores the data in smaller files which are end-to-end encrypted with bit protection. Developers can use a set of tools additionally coded in Keeper to protect their apps even more securly through crypto keys and locks. Locks are points which can be added inside your html form to help secure the data. Locks are un-breakable and can be onlu unlocked with a crypto key. Crypto Key's are unique keys because their passwords changes every 14 minutes. Password can be only retreived by accessing the crypto key section on the keeper portal online. But it does requies a crypto key to be present for the QR code inside the key. In order to obtain a lock and key, simply run the code using the pip command inside your terminal of your project. pip keepergetkey which will provide you with a 14 digits unique code which is the crypto key, lock can be added inside forms or any other element using {{keeper.addlock}}

Creating package for your web app after keeper has been installed is slightly hard since it requires around 3 hours to compile properly. Although it can be rushed by forcing some packages to install quickly but it is not recommended Keeper can be compiled inside your web application using simple command, pip keepercompileproject, which compiles the entire keeper inside your project.
